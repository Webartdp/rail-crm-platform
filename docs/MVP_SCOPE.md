# MVP Scope

## Main goal

Create the first usable version of the CRM platform for workforce and field operation management.

The MVP should solve the core operational problem first: who works where, when work started, when work stopped, what must be approved and what can later be billed.

## Included in MVP

### Users and access

- login
- roles
- permissions
- employee profile
- active / inactive users

### CRM core

- clients
- client contacts
- objects
- object locations
- assignments

### Field operations

- employee assignment list
- work start
- work stop
- travel event
- break event
- current employee status
- location snapshot if available

### Time tracking

- generated timesheet data
- manual correction request
- employee confirmation
- coordinator review
- manager approval

### Management

- daily view
- employee status list
- assignment filters
- approval queue
- audit log

## Not included in first MVP

- native mobile application
- complex accounting integration
- advanced KPI
- public customer portal
- full billing automation
- AI features

## Product rule

The MVP should be simple, stable and clear. It is better to implement fewer modules correctly than to copy every old MODX feature at once.
